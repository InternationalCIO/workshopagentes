Guía paso a paso · para Carlos Ruiz

Reproducir el workshop, de cero a producción

Esta guía es para cualquier persona — técnica o no técnica — que quiera recrear de principio a fin la web del workshop. Vamos en cuatro fases, en el mismo orden en que lo construimos en clase, sin dar nada por hecho.

Tiempo total estimado: 3 a 4 horas · Coste a final de mes: prácticamente cero.

1. Inventario: lo que necesitas antes de empezar

No necesitas saber programar. Sí necesitas tener instaladas y dadas de alta unas pocas cosas. Te las listo y te digo cómo conseguir cada una.

1.1 Cuentas online (gratis las tres)

  • GitHub — para guardar el código. github.com → "Sign up".
  • Anthropic Console — para crear la "llave" (API Key) que permite hablar con Claude. console.anthropic.com. Te pedirán método de pago: la primera llave viene con $5 de crédito gratis, suficiente para hacer todo este workshop diez veces.
  • Railway — para poner el "backend" en internet. railway.com → "Login with GitHub". El plan gratis sobra.

1.2 Programas en tu PC

  • Visual Studio Code (editor de texto, gratis) — code.visualstudio.com. Es donde verás los archivos. No necesitas saber usarlo, solo abrir carpetas y mirar archivos.
  • Git (control de versiones) — git-scm.com/downloads. Cuando lo instales, deja todas las opciones por defecto.
  • Node.js (entorno para JavaScript del servidor) — nodejs.org. Elige la versión "LTS" (la verde). Esto se necesita solo en la Fase IV.
  • Claude Code (la herramienta clave del workshop) — instalación con un comando, te lo digo abajo.
¿Por qué Claude Code y no ChatGPT u otro? Claude Code no es un chat: es un agente que escribe código directamente en tu carpeta. Tú le hablas en español, él modifica los archivos. Ahorra horas. Otros equivalentes (Cursor, Aider, etc.) hacen lo mismo, pero esta guía está hecha con Claude Code.

1.3 Instalar Claude Code

Abre una "terminal" (en Windows: busca "PowerShell" o "Terminal" en el menú inicio; en Mac: "Terminal"). Pega esto y pulsa Enter:

npm install -g @anthropic-ai/claude-code

Luego, para arrancarlo dentro de una carpeta, navegas hasta esa carpeta y escribes:

claude

La primera vez te pedirá iniciar sesión con tu cuenta de Anthropic.

2. Conceptos básicos (sin tecnicismos)

2.1 ¿Qué es un "agente"?

Un agente es un trozo de software con una responsabilidad clara. No necesariamente usa IA. Un termostato es un agente: mide la temperatura y enciende la calefacción. En este workshop construiremos cuatro agentes, en orden:

  1. orquestadorAgent — recibe lo que escribes y decide quién contesta. Es el "recepcionista".
  2. MatematicasAgente — sabe sumar, restar, multiplicar y dividir. Es el "contable".
  3. modeloAgente — habla con un modelo de IA real (Claude Haiku) para contestar preguntas. Es el "experto".
  4. corpusAgente — fabrica el cuaderno de información que usa modeloAgente. Es el "archivero".

2.2 ¿Cómo es la conversación con Claude Code?

Le hablas en lenguaje natural. Tú describes el qué quieres y él se encarga del cómo. Ejemplos:

«Hazme una página web con 16 tarjetas, una por participante, estilo MIT (rojo discreto, tipografía limpia). En cada tarjeta: foto opcional, nombre, rol y bio corta. Que sea responsive.»

Si algo no te gusta, se lo dices: «El rojo está demasiado oscuro» o «No me convencen las animaciones al pasar el ratón». Lo cambia. Esto es la idea fundamental del workshop: programar en español, no en inglés con paréntesis.

2.3 ¿Qué es "subir a GitHub"?

GitHub es como Dropbox para código. Tu carpeta vive en tu PC, pero también tienes una copia online (un "repositorio") sincronizada. Cada vez que cambias algo, lo "subes" (git push) y queda guardado online. Bonus: GitHub puede publicar gratis una web simple a partir de tu repo (eso se llama GitHub Pages).

3. Fase I — La web estática

~1 hora

El objetivo: una galería de participantes accesible desde una URL pública. Sin agentes todavía, sin IA. Solo HTML, CSS y un poco de JavaScript.

Paso 1.1 · Crear el repositorio en GitHub

Entra en github.com, botón "New" arriba a la izquierda. Nombre del repo: workshopagentes (o el que quieras). Marca Public. Marca también "Add a README file" (para que no empiece vacío). Click en "Create repository".

Paso 1.2 · Clonar el repo a tu PC

En la página del repo recién creado, botón verde "Code" → copia la URL HTTPS (algo como https://github.com/tu-usuario/workshopagentes.git).

Abre una terminal en una carpeta donde quieras tenerlo (por ejemplo, tu carpeta de Documentos). Escribe:

git clone https://github.com/tu-usuario/workshopagentes.git

Entra en la carpeta recién creada:

cd workshopagentes

Paso 1.3 · Arrancar Claude Code en esa carpeta

Desde la terminal, estando dentro de la carpeta del repo:

claude

Te abre un chat. Aquí escribirás los prompts (instrucciones en español).

Paso 1.4 · Pedir la primera versión

Pega este prompt (adáptalo a tus participantes):

Hazme una página web en HTML + CSS + JavaScript vanilla (sin frameworks, sin servidor) con una galería de tarjetas, una por participante de un workshop. Estilo MIT: color principal rojo #A31F34, tipografía 'Segoe UI', diseño limpio y responsive. Cada tarjeta tiene: avatar (la inicial del nombre como fallback), nombre, rol, biografía corta y un enlace opcional a LinkedIn. No incluyas datos personales (emails, teléfonos). Crea también una página "explicacion.html" donde cuentes en lenguaje no técnico cómo se construyó. Pon mis participantes a partir de la información que te paso a continuación: [pega aquí las bios]

Claude Code te hará preguntas o te enseñará lo que va a crear antes de hacerlo. Confírmale.

Paso 1.5 · Ver el resultado en tu navegador

Cuando termine, abre la carpeta del repo en el explorador de archivos, busca index.html y haz doble clic. Se abre en tu navegador. No necesitas servidor.

Paso 1.6 · Iterar hasta que te guste

¿Algo no encaja? Vuelve a Claude Code y díselo en español:

«El color rojo está demasiado intenso, suavízalo un poco. Y haz que las tarjetas tengan un efecto hover sutil que las eleve.»

Refresca el navegador y verás los cambios.

Paso 1.7 · Subir los cambios a GitHub

En la terminal (puede ser otra ventana, no hace falta cerrar Claude Code):

git add .
git commit -m "Fase I: galería de participantes"
git push origin main

La primera vez te pedirá usuario y contraseña de GitHub (o que abras el navegador para autorizar).

Paso 1.8 · Activar GitHub Pages

En la web de GitHub, dentro de tu repo: Settings → Pages. En "Source" elige "Deploy from a branch", branch main, folder / (root). Click "Save". En 1-2 minutos te dará una URL pública del tipo https://tu-usuario.github.io/workshopagentes/.

4. Fase II — Primer agente (smalltalk)

~30 minutos

El objetivo: añadir un widget de chat en la esquina inferior derecha. Cuando le hablas, te contesta con frases predefinidas según la "intención" del mensaje. Sin IA todavía, solo reglas.

Paso 2.1 · Pedir el widget de chat

En Claude Code (sigue abierto), pega:

Añade a la web un widget de chat flotante en la esquina inferior derecha (botón rojo redondo). Al abrirlo, aparece una ventana con un mensaje de bienvenida y un campo de texto. Crea un agente JavaScript llamado "orquestadorAgent" que vive en el navegador. NO usa IA: detecta la intención del mensaje del usuario buscando palabras clave (saludo, despedida, agradecimiento, "cómo estás"). Si reconoce alguna, contesta con una frase de un banco predefinido. Si el mensaje parece basura (sin vocales, letras repetidas), lo dice educadamente. Hazlo todo en español, sin emojis. Crea también una sección en explicacion.html que cuente esto en lenguaje no técnico, con la analogía de "el recepcionista que solo conoce 4 frases".

Paso 2.2 · Probar

Refresca el navegador. Click en el botón rojo abajo. Prueba: "hola", "gracias", "adiós", "asdfgjklñ". Cada uno debería disparar una respuesta distinta.

Paso 2.3 · Subir

git add .
git commit -m "Fase II: orquestadorAgent con smalltalk"
git push origin main

5. Fase III — Segundo agente (matemáticas)

~30 minutos

El objetivo: añadir un segundo agente que sepa hacer cuentas, y enseñar al orquestador a delegar en él cuando proceda. Aquí aparece el concepto clave de sistema multiagente.

Paso 3.1 · Pedir el segundo agente

Añade un segundo agente llamado MatematicasAgente, en un archivo separado (js/agente-matematicas.js). Solo sabe hacer cuatro cosas: /sumar A B, /restar A B, /multiplicar A B, /dividir A B, y un comando /ayuda. Si el orquestador detecta que el usuario empieza el mensaje con uno de esos comandos, delega en MatematicasAgente. Si no, sigue su lógica de smalltalk de antes. Maneja el caso "dividir entre cero" con un mensaje claro.

Paso 3.2 · Probar comandos

En el chat: /sumar 5 3 → debe responder 8. /dividir 10 0 → te avisa que no se puede.

Paso 3.3 · Mejora: lenguaje natural

Los comandos con barra son rígidos. Mejorémoslo:

Añade a MatematicasAgente la capacidad de detectar peticiones matemáticas en lenguaje natural. Reconoce sinónimos: "más"/"sumar"/"+", "menos"/"restar"/"-", "por"/"multiplicar"/"x"/"*", "entre"/"dividir"/"partido"/"/". Extrae los dos primeros números del mensaje. Si encuentra una operación Y al menos dos números, ejecuta el cálculo. Ejemplos que deben funcionar: "quiero sumar 5 y 3", "multiplica 4 por 6", "10 entre 2".

Paso 3.4 · Subir

git add .
git commit -m "Fase III: MatematicasAgente + lenguaje natural"
git push origin main

6. Fase IV — Tercer agente con IA real

~90 minutos

El objetivo: cuando el orquestador no sepa qué contestar (ni smalltalk ni matemáticas), delega en un modeloAgente que habla con Claude Haiku 4.5. Para mantener la API Key segura, este agente vive en un servidor (Railway). Aquí aparecen tres conceptos clave: RAG con corpus, cacheo y coste por consulta.

Antes de empezar, una regla de oro: a partir de aquí vas a manejar una API Key (una contraseña). Nunca la pegues en un chat, en un email, en una captura, ni la subas a GitHub. Si por error lo haces: revócala inmediatamente y crea otra. Ver sección 7 (Seguridad).

6.1 Crear la API Key de Anthropic

Entra en console.anthropic.com → Settings → API Keys → Create Key. Nombre: workshop-mit. Cópiala (empieza por sk-ant-api03-) y pégala en un gestor de contraseñas. No se puede ver dos veces.

6.2 Crear el segundo repositorio (para el backend)

En github.com → "New repository". Nombre: agenteworkshop. Marca Public (o privado, da igual). Marca "Add a README". Esto será el repositorio del servidor, separado del front.

¿Por qué dos repos? Porque el front vive en GitHub Pages (es solo HTML que cualquier navegador puede leer). Pero la API Key de Anthropic no puede estar en el front — sería pública. Por eso necesitas un servidor (el back) donde la Key esté como variable de entorno, oculta. La forma más limpia es tenerlos en repos separados.

6.3 Pedirle a Claude Code el backend

Vuelve a la carpeta del front (no clones el back todavía — Claude Code lo va a montar dentro como subcarpeta para no perderse). En Claude Code:

Crea una subcarpeta llamada "agenteworkshop" (añádela al .gitignore del front para que los dos repos no se mezclen). Dentro, monta un backend con Node.js + Express, listo para desplegar en Railway. Endpoints: GET / (ping de salud), GET /info (info de agentes sin secretos), POST /consultar (recibe un mensaje, llama a modeloAgente y devuelve la respuesta). La API Key vive solo en .env (gitignored) con la variable ANTHROPIC_API_KEY. Crea dos agentes: modeloAgente (llama a Claude Haiku 4.5 usando el SDK oficial @anthropic-ai/sdk, con el corpus como system prompt y cache_control ephemeral, y devuelve también una estimación del coste de cada llamada en USD según las tarifas de Haiku: Input $1/MTok, Output $5/MTok, Cache Write $1.25/MTok, Cache Read $0.10/MTok) y corpusAgente (lee corpus/corpus.md y lo proporciona al modeloAgente; si no existe, lanza un error CORPUS_MISSING que el modeloAgente capture para responder al usuario con una frase humorística tipo "estoy ocupado, vuelve más tarde"). Crea también un README.md y un .env.example.

6.4 Generar el corpus

El corpus es el "cuaderno" del agente: contiene todo lo que sabe (en este caso: los 16 participantes, el programa del workshop y cómo se construyó la web). Tienes dos opciones:

  • Manual: escribir un archivo Markdown estructurado con la info de cada participante (rol, experiencia, empresas, formación, curiosidades).
  • Asistido: dejar a Claude Code que lo extraiga de PDFs/Word con las librerías pdf-parse y mammoth. Pídeselo así:
Quiero generar el corpus a partir de los CVs originales que están en la carpeta CORPUS/ (formatos .txt, .pdf, .docx, uno por participante). Crea un script que extraiga el texto, te muestre el contenido, y a partir de él construye un corpus.md compacto y bien estructurado: encabezado con reglas para el agente ("solo responde a partir de este documento"), sección "El programa", sección "El facilitador", sección "Los 16 participantes" (cada uno con rol, experiencia, empresas, formación, curiosidades) y sección "Cómo se construyó la web". Guárdalo en agenteworkshop/corpus/corpus.md. Los CVs originales NO se suben a GitHub (mantén CORPUS/ en el .gitignore).

6.5 Subir el backend a su propio repo

Desde la subcarpeta agenteworkshop (en otra terminal):

cd agenteworkshop
git init
git branch -M main
git remote add origin https://github.com/tu-usuario/agenteworkshop.git
git add .
git commit -m "Fase IV: backend con modeloAgente y corpusAgente"
git pull --rebase origin main # solo la primera vez, si el repo tenía README
git push -u origin main

Si hay conflicto en el README (por el que vino con el repo), resuélvelo manteniendo el tuyo y haz git rebase --continue.

6.6 Desplegar en Railway

En railway.com:

  1. "New Project" → "Deploy from GitHub repo".
  2. Importantísimo: elige el repo agenteworkshop (¡NO el del front!).
  3. Railway detecta que es Node y empieza a construir (~1 minuto).
  4. Pestaña Variables → "New Variable":
    • Nombre: ANTHROPIC_API_KEY
    • Valor: tu clave de Anthropic (sin espacios, sin comillas).
  5. Railway redesplega automáticamente con la variable cargada.
  6. Pestaña Settings → Networking → Public Networking → Generate Domain. Target port: 8080. Click "Generate Domain".
  7. Te da una URL pública. Cópiala y ábrela en el navegador. Debes ver:
    {"servicio":"agenteworkshop","estado":"activo","fase":"IV",...}
Si ves un 401 "invalid x-api-key": la clave que pegaste en Variables está mal, vacía o tiene espacios. Comprueba (icono del ojo 👁) y, si hace falta, genera otra nueva en console.anthropic.com.

6.7 Conectar el front al back

En Claude Code, en la carpeta del front:

En el front: cuando el orquestador no pueda resolver con smalltalk ni con matemáticas, que delegue en el backend haciendo fetch POST a /consultar. La URL del backend la inyectamos desde index.html con window.AGENTE_WORKSHOP_BACKEND_URL = '[URL de Railway aquí]'. Mientras espera, muestra un indicador de "escribiendo..." (tres puntos animados). Cuando llegue la respuesta, muéstrala junto a un pequeño desglose del coste en USD (tokens de input, output y cache). Si el backend falla, muestra un mensaje claro al usuario.

Después: git add . → git commit → git push. GitHub Pages tarda 1-3 minutos en actualizar.

6.8 Probar todo el flujo

Abre la URL pública del front (tu GitHub Pages) y abre el chat. Prueba:

  • Saludo: «hola» → contesta el orquestador.
  • Matemática: «suma 5 y 3» → contesta MatematicasAgente.
  • Pregunta sobre el corpus: «¿Quién es Cecilia?» → contesta modeloAgente con Haiku 4.5 + desglose del coste.
  • Pregunta fuera del corpus: «¿Cuál es la capital de Mongolia?» → debe responder honestamente que no lo sabe.

6.9 Bonus: respuesta humorística sin corpus

Para enseñar a la clase qué pasa si el corpus desaparece, renombra el archivo:

cd agenteworkshop
git mv corpus/corpus.md corpus/corpus.md.disabled
git commit -am "TEST: simula falta de corpus"
git push

Railway redesplega en 30 seg. A partir de ese momento, cualquier pregunta que no sea smalltalk ni cuenta recibirá una de las frases humorísticas ("estoy ocupado, vuelve más tarde", etc.). Para revertir: rename inverso.

7. Seguridad: las reglas que no se rompen

1. La API Key NUNCA se enseña. Ni en pantalla, ni en chats, ni en capturas, ni en presentaciones, ni en un Google Doc compartido, ni en un email. Si ha "salido" del sitio donde debía estar — considérala comprometida.
2. La API Key NUNCA se commitea. Vive solo en .env. Y .env está en .gitignore. Comprueba siempre con git status antes de hacer commit que no aparece ningún archivo sospechoso.
3. Si la clave se filtra, revócala en 60 segundos. Anthropic Console → Settings → API Keys → "Delete". Genera otra. Actualiza Railway.
4. El front no puede llamar directamente a Anthropic. Todo el tráfico hacia el modelo pasa por el backend. Esta separación cliente/servidor es la práctica estándar de producción y no se puede saltar — por eso usamos Railway.

8. Cierre y siguientes pasos

Al llegar aquí tienes una web pública con cuatro agentes, un modelo de IA real, un corpus curado, cacheo y cálculo de coste por consulta. Y dos repositorios bien organizados, listos para seguir creciendo. Lo más importante: lo has hecho describiendo lo que querías en español, sin escribir tú mismo la mayoría del código.

¿Por dónde seguir?

  • Más agentes: uno de traducción, uno de calendario, uno de búsqueda web... cada uno pequeño y especializado.
  • Multi-modelo: el modeloAgente ya está preparado para cambiar entre Haiku/Sonnet/Opus según el tipo de pregunta.
  • Memoria persistente: guardar la conversación en una base de datos para que el agente recuerde entre sesiones.
  • Tools / function calling: permitir que el modelo llame a APIs externas (clima, vuelos, etc.) durante la conversación.
  • MCP: el protocolo que permite que un agente use herramientas estándar (calendario, correo, drive) sin escribir integraciones a medida.
Para Carlos, con cariño.
Construir cosas es la mejor forma de entenderlas. Si llegas hasta aquí siguiendo cada paso, no es que —además— hayas aprendido a usar Claude Code y a desplegar un agente: es que habrás demostrado que la barrera de entrada a la IA agéntica es mucho más baja de lo que parece. La parte técnica se aprende. La intención —qué construir, para quién, con qué cuidado— esa siempre es tuya.
